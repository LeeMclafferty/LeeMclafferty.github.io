CPU Renderer

Welcome to my portfolio project all about 3D graphics. I'm a student software developer, and alongside my schoolwork, I've been getting into the world of 3D graphics. I started off by digging into this book called "Computer Graphics from Scratch." No graphics API here, just raw graphics. I've always been fascinated by computer graphics, so I thought, "Hey, let's learn the nitty-gritty before diving into the fancy graphics APIs."

My project uses a bit of SDL for window creation, rendering, and displaying an SDL_Texture. Outside of that I am manipulating everything on an array that represents the pixel size of the window. With this setup, I crafted some functions to rasterize pixels, draw lines, and triangles. Right now, the renderer can read .OBJ files, that are given a path in the code. My goal is to add more features, like moving objects in the viewport, importing OBJs with a simple GUI, and of course, refactoring and optimizing.
