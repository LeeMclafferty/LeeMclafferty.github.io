Software Renderer

My current "work in progress" project is a software-based renderer. I embarked on this project to grasp the fundamentals of computer graphics. As it progresses, I feel the project wonderfully fulfills this aim. How deep should one delve into the basics? I'm convinced that demystifying the "black box" around many computer science topics boosts my confidence, enabling me to undertake larger projects. When confronted with an error from a graphics API, understanding the underlying problem becomes easier if you've previously implemented your version. With this project, I delved deep, even utilizing SDL for pixel-by-pixel math operations. This allowed me to manually implement features that are often taken for granted, such as drawing lines, filling faces, implementing lighting, and more.

I initiated the project by establishing a foundation in linear algebra. Subsequently, I implemented Vector2D, 3D, and 4D structures to carry out operations. Then, matrices were added to apply transformations to geometry. While I could delve into myriad details, I've compartmentalized the project into distinct phases, each meticulously tracked and planned on a Milanote board. The first phase, research, involved brushing up on linear algebra, acquainting myself with programming patterns I aspired to use, and studying common techniques found in CPU-based rendering systems.

In Phase 2, I transitioned to setting up SDL and strategizing the project's structure. I opted for a predominantly object-oriented approach for this renderer. While debates about OOP versus other programming paradigms are endless, I firmly believe that OOP fosters a deeper understanding of mechanisms. Designing each class allows me to meticulously consider how components interrelate and discern strategies to decouple them, ensuring robust software.

During Phase 3, I focused on rendering 3D geometry. While I began with 2D geometry in the first two phases, it primarily laid the groundwork for mastering manual drawing without SDL functions. Phase 3 marked my foray into importing triangulated .obj files, displaying them on-screen, and introducing transformations. Innovations included perspective projection, the painter’s algorithm, backface culling, and more.

Phase 4 is ongoing, a segment I revisit as necessary. As of now, it's concentrated on studying the perspective divide to optimize meshes, ensuring they exclusively use the viewing plane for rendering.

Currently, I'm navigating the fifth phase, which has been exhilarating. My attention is primarily on enhancing the lighting within my engine. I've successfully implemented a type of global illumination for flat shading, using bit-shift operations to deduce the face color based on the lighting. Presently, I'm delving into a smooth shading solution. My investigations have introduced me to both Phong and Gouraud Shading. Given the performance intricacies of Phong shading — seemingly more apt for GPU tasks — I've decided to proceed with Gouraud Shading.
